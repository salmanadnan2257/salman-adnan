How I build

Durability proven, not claimed.

  1. 01

    Scope it end to end

    I write the tests early, so the thing that ships is proven under failure, not hoped to work.

  2. 02

    Build the hard core from scratch

    Storage engines, consensus, inference. I build the difficult part myself so I actually understand it.

  3. 03

    Measure honestly

    I report the real numbers, including the ones that make a feature look worse. Speculative decoding was slower on CPU, so I said so.

  4. 04

    Document so it runs

    Every project ships with a README and tests someone else can run without me in the room.

Correct under failure, or it does not count.
